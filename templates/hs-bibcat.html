Haskell BIBCAT

In the preliminary stages of a functional programming project for BIBCAT being developed in Haskell. The inspiration for this work is the Pandoc project by John MacFarlane that allows you take different document formats such as docx, markdown, html, epub and convert them into other formats including PDF, docx, markdown, epub, and markdown among others.

  • Haskell has strong graph data structure support built into the language, much of what we're attempt to do with BIBCAT is at a high-level graph manipulation and translation between different graphs. Moving towards a pure graph implementation will hopefully simplify and speed-up some of the operations in BIBCAT
  • Haskell programs can be complied that along with its graph support, should provide a significant speed boost to the ingester processing from different metadata formats to BIBFRAME and Schema.org RDF
  • Haskell has influenced important features of Python including decorators and now with the release of Python 3.6, Python now has optional typing that is very similar to Haskell's strong typing support.

The Catalog Pull Platform has two active Haskell projects:

  • A fork of abandoned project on Google Code, the hsmarc project available at https://github.com/KnowledgeLinks/hsmarc provides operations on MARC21 records.
  • The bibcaths project, available at https://github.com/KnowledgeLinks/bibcat-hs is still in it's infancy but the features set will initially include support for converting MARC21, XML, and CSV into BIBFRAME RDF using the same RDF Rules used in the BIBCAT project.
